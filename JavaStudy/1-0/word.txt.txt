■ IT
IT－Information Technologyの略で、日本語では「情報技術」と訳される。
コンピュータやインターネットの通信に関する技術の総称。

ICT－Information and Communication Technologyの略で、日本語では「情報通信科学」と訳される。
「IT」とほぼ同義で、ITが情報技術そのものを指すのに対し、ICTは情報の伝達方法、活用方法などを指す。

IoT－Internet of Things の略で、日本語では「モノのインターネット」と訳される。
あらゆるモノに通信機能を持たせ、インターネットに接続する技術のことを指す。

■ OS
OS－Operating　Systemの略で、基本ソフトのことを指す。
コンピュータ全体の管理、制御し、様々なアプリケーションを動かすための最も基本的なソフトウェアである。

Windows－Microsoft社が開発・販売しているOS製品のシリーズ名。

macOS (Mac OS X)－Apple社がパソコン製品の「Mac」シリーズ向けに提供しているOS製品。

UNIX（ユニックス）－OSの一種で、現存している中では最も古いOSのひとつ。
このUNIXという名称は商標であり，現在権利を有している The Open Group が認めたものはUNIXという名称を使用できる。

Linux（リナックス）－世界で最も普及しているUNIXライクなオープンソースのOS。
プログラムが公開されており誰でも自由に入手や使用、改変、再配布することができる。

■ 言語・環境
マークアップ／フロントエンド－マークアップとは、webサイトを作成する時に「タグ」と呼ばれる目印を使ってサイト内の文書に意味づけをしていく作業のこと。
フロントエンドとは、直接ユーザーの目に触れる部分のことで、クライアントサイドとも呼ばれる。

HTML－HyperText Markup Languageの略で、Webページを記述するためのマークアップ言語。
「タグ」と呼ばれるもので囲うことにより、文章の構造や表示の仕方などを文書に埋め込んで記述することができる。

CSS－Cascading Style Sheets（カスケーディング・スタイル・シート）の略で、Webページのスタイルを指定するための言語。
HTMLと組み合わせて使用し、HTMLが定義した情報構造などをどのように装飾するかを指定する。

JavaScript－Webページに複雑な機能をつけることができる言語で、動的なWebページを作ることができる。
HTML・CSSを組み合わせて使用する。


サーバーサイド－サーバー側（ユーザから見えない場所）でプログラムの実行・管理したりすること。

.NET（C#, VB）－「.NET」とは、正式には「.NET Framework」と呼ばれ、Microsoft社が提供しているアプリケーション開発、実行環境のこと。
C#とVB( Visual Basic）は、.NET Framework 上で実行されるさまざまな種類のアプリケーションを作成するために設計されたオブジェクト指向型のプログラミング言語。

Java－様々な分野で使用されているオブジェクト指向型のプログラミング言語。
特定のコンピュータの機種やOS依存することなく、基本的にはどのようなプラットフォームでも動作することが特徴。

Java SE－Java Platform, Standard Editionの略で、「Java」における基本的な「API」などをまとめた機能セットの一つ。
（API－Application Programming Interfaceの略で、Java の機能やデータなどを利用するための呼び出し方を定義したもの。）

Java EE（Jakarta EE）－Java Platform, Enterprise Editionの略で、「Java」における基本的な「API」などをまとめた機能セットの一つ。
標準機能セットのJava SEの拡張として提供され、主にWebアプリケーション構築や、大規模開発のための機能が定義されている。

Perl－Practical Extraction and Report Languageの略で、テキスト処理が得意なスクリプト言語。UNIX環境で発達してきた。
文字列と数値を自動的に変換し、型を意識せずコードを書くことができる。可読性が高く、インプリタ型言語なのでコンパイルが不要。

PHP－Web開発によく使用され、HTMLに埋め込み動的なページを作ることができるスクリプト言語。
MySqlやPostgerSQLなどさまざまなデータベースと連携したWebサイトを簡単に作成することができる。
同じスクリプト言語であるjavaScriptとの大きな違いとして、JavaScriptはクライアントサイドがコードを受け取りそこで動作するため、コードが誰でも見れてしまうのに対し、
PHPはサーバーサイドで実行しクライアントサイドはその結果のみを受け取るため、どのようなコードなのかを見ることはできないという点がある。

Ruby－日本人により開発されたオブジェクト指向型のプログラミング言語で、コンパイルが不要なインプリタを採用したスクリプト言語の一種。
書きやすく、読みやすく、シンプル。
Ruby　on RailsというWebアプリケーションフレームワークを使用することができる。 

■ サーバーソフト
Apache（アパッチ）－Apache HTTP Server。 世界的に最も普及しているWebサーバ（HTTPサーバ）ソフトウェアの一つで、様々なOSで動作する。
Apache Software Foundationが開発しており、オープンソースソフトウェアとして公開している。
PHPやデータベースと連携がしやすく、モジュールにより基本機能を拡張可能。

nginx－「エンジンエックス」と発音する。オープンソースのWebサーバソフトウェアの一つ。
高速で大量処理が得意（大量の動的コンテンツはApacheの方が得意）。

IIS－Internet Information Servicesの略で、Microsoft社のWindows Serverシリーズに同梱されているWebサーバソフトウェア。
視覚的にマウス操作ができるので比較的難易度は優しい。IISならではの機能として、ASP.NETがあり、高機能なWebアプリの開発ができる。


■ SQL
SQL（シークェル）－Structured Query Languageの略で、データベースの管理や操作を行うためのデータベース言語の一つ。
広く普及しており、様々なデータベース管理システム（DBMS：Databese Management System）で利用できる。
プログラミング言語を利用するなかで、データベースに対する操作が必要になった場合は、SQL文を使う。

DB2－IBM社のRDBMS製品のこと。

MySQL－オープンソースのRDBMSの一つで、現在はオラクル社が開発を主催している。
現在世界ではオープンソース系のDBとしてはディファクトスタンダードに近い。
WordpressをはじめとしたCMS(コンテンツ・マネジメント・システム)に積極的に採用されたことからWeb系の定番に。

Maria DB－「My　SQL」から派生したオープンソースのRDBMS。
ビッグデータなどの集計作業が得意で、高機能且つ使い勝手が良い。

Oracle DB－オラクル社が主催しているRDBMS。
高い信頼性と移植性、堅牢性などを備えているが高価。Oracle　DB技術者の認定資格もある。

PostgreSQL（ポストグレスキューエル）－フリーのオープンソースのRDBMS。
Unix系や主要なOSに対応し、バランスが良く機能が豊富である。

SQLite－パブリックドメイン（著作権放棄）のオープンソースのRDBMS。
多くのDBMSと異なり、サーバーにインストールせずに開発用ライブラリとしてソフトウェアに組み込んで使用することができる。

SQL Server－Microsoft社が主催しているRDBMS。
Windows及びLinuxで動作し、Windows　Server環境で構築された業務システムのデータ管理に採用されることが多い。

CRUD（クラッド）－「Create/Read/Update/Delete」の略で、主にDBMSについて用いられる語。
この４つに対応する機能を実装することでデータの操作や管理を行うことができ、ソフトウェアなどの設計の際にはCRUD図を作成することがある。

トランザクション/transaction－ITの分野では、ソフトウェアの処理方式の一つで複数の処理を一つにまとめたもの。
一体として管理された処理は全て「成功」か「失敗」かのいずれかのみ。
（複数の処理は分離することができない。）

コミット/ commit－ITの分野では、処理や変更などを確定させる、反映させる、といった意味で用いられることが多い。
バージョン管理システムにおいては更新したファイルの内容をリポジトリに反映させる処理のことを「コミット」という。

ロールバック－システム障害などが起きた際に、正常に稼働していた時点の状態に戻して復旧を試みることを指す。
トランザクション処理でよく用いられる。

排他制御（排他ロック）－ダブルブッキングしないようにするために、
それぞれの主体の利用の制限や禁止を行う仕組みのこと。
主に複数の人やプログラムで共有されるデータを扱う際に必要とする。

楽観ロック－めったに他者との同時更新は起きないであろう、という楽観的な前提の排他制御。
データにロックはかけず、更新対象のデータがデータ取得時と同じ状態であることを確認してから更新することで、
データの整合性を保証する方式。

悲観ロック－他者が同じデータに頻繁に変更を加えるであろう、という悲観的な前提の排他制御。
更新対象のデータを取得する際にロックをかけることで、他のトランザクションから更新されないようにする方式。

実行計画－SQLを効率的に実行するための計画のことで、主にテーブルの結合方法やデータ取得方法などが書かれている。
データベースの実態を反映する「統計情報」を基に作成する。

チューニング/tuning－ITの分野では、情報システムやコンピュータなどの設定や構成を調整し、
目標の状態に近づけたり、性能を最大限引き出したりする作業を指すことが多い。

パフォーマンス/performance－ITの分野では、コンピュータなどの機器やソフトウェア、
システムなどの処理性能や実行速度、通信回線・ネットワークなどの伝送速度・容量などのことを指すことが多い。

スロークエリ－データベースにおいて、時間のかかっているSQL(遅いSQL)のこと。
基本的には、MySQLやPostgreSQLの設定により取得することができ、どの程度遅いものを対象とするか等を指定することができる。

クエリログ－MySQLが出力するログ。
問題が発生した際に原因を解明する際や、障害からの復旧の際にもログが重要になる。
エラーログ/一般クエリログ/バイナリログ/リレーログ/スロークエリログなどがある。

IDE（統合開発環境）－Integrated Development Environmentの略で、テキストエディタ、コンパイラ、デバッガなどを1つにまとめ、
一つの開発環境で統合・統一的に扱えるようにした開発環境。

Eclipse－Javaをはじめ、多くの言語に対応しているオープンソースのIDEの一つ。

NetBeans－Javaアプリケーション開発のIDEとして開発が始められ、現在では多くの言語に対応しているオープンソースのIDEの一つ。
プロジェクト管理機能や入力支援機能を備え、GUIによるプログラム開発が手軽にできる。

SpringToolSuite－通称STS。Spring Framwrowkの開発元が提供しているEclipseをベースとして作成されたIDEで、
Spring Frameworkを利用した開発に適した機能が備えられている。

DBツール－データベース管理タスクに使用できるツール、ユーティリティおよび補助ツールの総称。

A5:SQL Mk-2－複雑化するデータベース開発を支援するために開発されたフリーのSQL開発ツール。
SQL文の入力支援やER図作成などの機能を備えた高機能なSQL開発環境。

pgAdmin－GUIによるPostgreSQLの管理ツール。

DBeaver－「DBeaver Corp」が開発したマルチプラットフォーム対応のSQLクライアントツール。
殆どのDBに対応している。

MySQL Workbench－データベースの設計・開発・管理を一本で実現している無料の統合ビジュアルツール。
GUIに対応しており、Windows、Linux、Mac OS X で利用可能。

Sequel Pro－Mac用のMySQLのデータをGUI操作で管理するための無料のアプリケーション。
GUIに対応しており、操作が簡単。

TeamSQL－マルチプラットフォーム対応のSQLクライアントツール。（サービス終了）
Windows、Mac、Linuxで使用可能。

■ 肩書
PM－Project Managerの略で、プロジェクトの計画・遂行に責任を負う管理者。
要件定義からリスク管理まで、プロジェクト全体を管理する。
組織によってはPLと読んだり、PMとは別にPL職を設置している場合がある。

PL－project leaderの略で、、プロジェクトを遂行するチームの責任者。
また、プロジェクトマネージャを補佐し、現場の業務の実施・遂行に責任を持つ監督者。
比較的大規模なプロジェクトではPMとPLが両方置かれることが多い。

PG（PE）－programmerの略で、プログラミング言語やソフトウェア開発ツールなどを用いてコンピュータプログラムを作成・修正する技術者のこと。

SE－System Engineerの略で、情報システムの企画、設計、開発、試験、構築、導入、運用、更新、修正、廃棄などに携わる技術者の総称。
組織によって意味が変わるが、主に上流工程で要件定義、設計などを担当する技術者のこを指すことが多い。

デザイナーIT業界においては、WebサイトのデザインをするWebデザイナーを指すことが多い。

エンジニア－技術者のことを指し、IT分野では対象が広く様々な種類がある。
PM、SE、PG、サーバー/ネットワークエンジニア、セキュリティエンジニアなど。

フロントエンド・エンジニア－ユーザ側の目に触れる「フロントエンド」側を担当するエンジニア。
Webデザイナーのデザインをもとに、HTML、CSS、JavaScript、PHPの設計や実装、WordPressなどのCMS（コンテンツマネジメントシステム）構築やカスタマイズを担当する。

サーバーサイド・エンジニア－サーバー側で動作するプログラムの開発やデータ処理などをおこなうエンジニア。
バックエンド・エンジニアとも呼ばれるが、バックエンド・エンジニアよりも広い範囲を指して使われることがある。。

バックエンド・エンジニア－フロント・エンドに対しWebサイトの裏側、サイト内部の処理部分などの「バックエンド」を担当するエンジニア。
サーバーの構築、DBの構築、システムの運用・開発などを行う。
よく使用するプログラミング言語はJava,PHP、Python、Rubyなど。

■ UI（ユーザーインターフェース）
UI－User Interfaceの略で、ユーザの目に触れ、使用する部分のことを指す。

CUI－Character User Interfaceの略で、UIがテキストデータのみで表現され、コマンドを入力して行うやり取りのこと。
主にUNIXやLinuxなどのOS、シスコ社製のルーターなど機器の操作・保守管理、SQL言語を用いたデータベースの操作などに使用されている。
Windowsに搭載されているコマンドプロンプトもCUI。入力はキーボードの操作が必要。

GUI－Graphical User Interfaceの略で、UIが画像やアイコンなどで表現され、直感的、感覚的に操作できる。
マウスだけで操作できることが特徴。


■ ファイル／ディレクトリ
ファイル－PCにおけるファイルとは、データそのもののことを指す。

拡張子－ファイルの種別を表す JPG、PDF、doc、mp3、exeなどのこと。

ディレクトリ－GUIではフォルダ、CUIではディレクトリと呼ぶことが多い。
ハードディスクなどの記録メディア上のファイルの保存場所。

ルート・ディレクトリ－階層構造の一番上にある場所のこと。

カレント・ディレクトリ－現在いるディレクトリのこと。

パス－ディレクトリやファイルの場所を表している。

絶対パス－ルーとディレクトリからの経路を表している。

相対パス－カレントディレクトリからの経路を表している。
ホームページの作成などでよく使われる。

■ ネットワーク
LAN－Local Area Networkの略で、限られた範囲内にあるコンピュータや通信機器などをケーブルなどで接続し、相互にデータ通信できるようにしたネットワークのこと。
主に室内、建物内程度の広さで構築されるものを指す。

WAN－Wide Area Network の略で、地理的に離れた地点間を結ぶ通信ネットワーク。
建物内や敷地内を結ぶLANと対比される用語で、通信事業者が設置・運用する回線網のことを指すことが多い。

クライアント－ITの分野では、他のコンピュータやソフトウェアから機能や情報の提供を受けるコンピュータやソフトウェアのことを指す。

サーバー－コンピュータネットワークにおいて、他のコンピュータに対し、自身の持っている機能やサービス、データなどを提供するコンピュータのこと。
また、そのような機能を持ったソフトウェア。

■ 各種プロトコル
HTTP－HyperText Transfer Protocolの略で、WebサーバとWebクライアントの間でデータの送受信を行うために用いられるプロトコル（通信規約）。
HTMLファイルや、ページに関連付けられたスタイルシート、スクリプト、画像、音声、動画などのファイルを、データ形式などのメタ情報を含めてやり取りすることができる。

FTP－File Transfer Protocolの略で、インターネットなどのTCP/IPネットワークでファイル転送を行うことができるプロトコルの一つ。
FTPサーバ、FTPクライアントの二種類のソフトウェアを用い、両者の間で接続を確立し、クライアントからの要求に基づいてファイルを送受信することができる。

HTTPS－ HTTP Secureの略で、通信プロトコルの種別などを表すURIスキームの一つ。
Webのデータ転送に用いられるHTTPが、SSLやTLSで暗号化されている状態を表したもの。
WebサーバとWebブラウザの間の通信が暗号化されていることを意味し、通信経路上での盗聴や第三者によるなりすましを防止する。

SSL－Secure Sockets Layerの略で、インターネットなどのIPネットワークでデータを暗号化して送受信するプロトコルの一つ。
データを送受信する一対の機器間で通信を暗号化し、中継装置などネットワーク上の他の機器による成りすましやデータの盗み見、改竄などを防ぐことができる。

SSH－Secure SHellの略で、主にUNIXコンピュータで利用される、ネットワークを介して別のコンピュータにログインして操作するためのソフトウェアの一つ。
通信経路が暗号化されるため、インターネットなどを経由しても安全にアクセスすることができる。

ポート、ポート番号－ITの分野では、ネットワークでデータ通信を行うための扉のようなもの。
一般的には、機器の筐体などに設けられたケーブルやコネクタの差込口のことをポートという。
ポート番号とは、部屋番号のようなもの。
インターネットで標準的に用いられるプロトコルであるTCP/IPにおいて、同じコンピュータ内で動作する複数のソフトウェアのどれが通信するかを指定するための番号。

HTTP、FTP、SSHの各ポート番号－TCP 80 : HTTP/TCP 20 : FTP (データ)/TCP 21 : FTP (制御)/TCP 22 : SSH

ファイアウォール－ネットワークの境界に設置され、内外の通信を中継・監視し、外部の攻撃から内部を保護するためのソフトウェアや機器、システムなどのこと。

TCP－ Transmission Control Protocol の略で、インターネットなどのネットワークで、IP（Internet Protocol）の一段階上位層のプロトコルとして標準的に使われるものの一つ。

■ IPアドレス
IPアドレス－インターネットにおける、場所のようなもので、通信機器一台ずつに割り当てられた重複しない識別番号。

グローバルIPアドレス－インターネット上で使用するアドレスのことで、世界に一意のIPアドレス。
アドレス発行組織に申請を行って割り当てを受けなければならない。

プライベートIPアドレス－特定の組織内ネットワークのみで通用するアドレスのことで、組織で一意のIPアドレス。
ローカルIPアドレスともいう。
各組織ごとに設置・運用されているLANなどのネットワーク上で用いられるアドレスで、申請などは不要。

■ ドメイン
ドメイン－人間に分かりやすいよう、IPアドレスに対応するアルファベットと数字などでつける別名。

DNS－Dmain Name Systemの略で、IPアドレスとドメイン名を関連付けるサーバ。

VPN－Virtual Private Networkの略で、、通信事業者の公衆回線を経由して構築された仮想的な組織内ネットワーク。また、そのようなネットワークを構築できる通信サービス。
企業内ネットワークの拠点間接続などに使われ、あたかも自社ネットワーク内部の通信のように遠隔地の拠点との通信が行える。

ブリッジ－ネットワークの分野で、複数のネットワークセグメントを結ぶ中継機器のうち、
受信したデータのMACアドレスなどデータリンク層の宛先情報を参照して中継の可否を判断する機能を持ったものをネットワークブリッジあるいは単にブリッジという。

プロキシサーバー－企業などの内部ネットワークとインターネットの境界にあり、内部のコンピュータの「代理」（proxy）としてインターネット上のコンピュータへ接続を行うコンピュータのこと。また、そのような機能を持つサーバソフトウェア。

■ 電子メール
SMTP－Simple Mail Transfer Protocol の略で、インターネットなどのTCP/IPネットワークで標準的に用いられる、電子メールを伝送するためのプロトコルの一つ。
メールサーバ間でシステム間で交わされるデータ形式や伝送手順などを決めている。

POP－Post Office Protocolの略で、電子メールで使われるプロトコルのひとつ。
ユーザーがメールサーバから自分のメールを取り出す時に使用する、メール受信用プロトコル。

IMAP－Internet Message Access Protocolの略で、メールサーバ上の電子メールにアクセスし操作するためのプロトコル。

■ セッションとクッキー
セッション－ITの分野では、コンピュータシステムやネットワーク通信において、接続/ログインしてから、切断/ログオフするまでの、一連の操作や通信のことをこのように呼ぶ。
あるコンピュータシステムに利用者や別のシステムがアクセスし、連続して交わされる一連の通信や操作の連なり。

クッキーHTTP Cookieのこと。Webサイトの提供者が、Webブラウザを通じて訪問者のコンピュータに一時的にデータを書き込んで保存させる仕組み。

キャッシュ－低速な記憶装置や伝送路から読み出したデータのうち、直近に読み込んだものや使用頻度が高いものを高速な記憶装置に複製しておくこと。

■ DB（データベース）について
インデックス－ITの分野では、書籍の巻末の索引のように、多数のものの中から特定の対象をすばやく見つけ出すため識別情報や、整列された所在情報の一覧などを指すことが多い。

■ テストについて
フロー－ITの分野では、コンピュータプログラムを構成する命令や手続き、処理の流れなどのことをフローという。
通信の分野では回線を通じて伝送される信号やデータの流れを「データフロー」あるいは単にフローという。
情報システム上での機器間やソフトウェア間のデータの移動や伝達のこともデータフローということがある。

■ チケット管理ツール
タスク－ITの分野ではソフトウェアの実行単位のことをタスクということがある。
メインメモリ上に展開され実行状態にあるプログラムのことを指すもので、「プロセス」とほぼ同義である。

■ バージョン管理
Git－オープンソースの分散バージョン管理システムの一つ。
複数の開発者が共同で一つのソフトウェアを開発する際などに、ソースコードやドキュメントなどの編集履歴を統一的に管理するのに用いられる。

SVN－Subversionの略で、著名なオープンソースのバージョン管理システムの一つ。複数の開発者が共同で一つのソフトウェアを開発する際などに、ソースコードやドキュメントを管理するのに用いられる。
CVS（Concurrent Versions System）を参考に開発され、基本的なコマンドなどはCVSと共通になっている。

デグレ－degrade（デグレード）のこと。新しいバージョンのソフトウェアの品質が、以前より悪くなること。また、以前修正した不具合やバグが再発・復活すること。
新しいファイルなどを古い内容で上書きしてしまい、更新内容が失われることを指す場合もある。

マージツール－バージョン管理システムなどで管理している開発中のプログラムなどに、新たにコードを追加したり、修正を反映させたりする操作をマージといい、
それを行うことのできるツールを指す。
